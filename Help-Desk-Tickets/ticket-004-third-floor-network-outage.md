# Ticket #004 - Third Floor Network Outage

## Ticket Information

**Priority:** High

**Affected Area:** Third Floor

**Primary Contact:** Mike Johnson

---

## Issue Description

Users on the third floor reported being unable to access the internet. The issue affected multiple departments and conference rooms. Users on other floors remained operational.

---

## Business Impact

Multiple users on the third floor were unable to access internet-based resources, impacting productivity across several departments.

---

## Initial Assessment

Because the issue affected an entire floor rather than a single user, a network infrastructure issue was suspected.

---

## Troubleshooting Steps

1. Gathered information from the reporting user.
2. Confirmed only the third floor was affected.
3. Verified the second floor remained operational.
4. Reviewed network documentation.
5. Investigated network infrastructure in the Server Room.
6. Identified the Floor 3 switch showing an ERROR status.
7. Reviewed network topology and confirmed all other infrastructure components were operational.
8. Notified the affected user of planned maintenance.
9. Rebooted the Floor 3 switch.
10. Verified the switch returned to an online state.
11. Confirmed with the user that internet connectivity was restored.

---

## Root Cause

The Floor 3 network switch entered an error state, disrupting internet connectivity for users connected to that switch.

---

## Resolution

Rebooted the affected Floor 3 switch and verified restoration of network services.

---

---

## Outcome

Internet access restored

User confirmed resolution

Ticket closed

---

## Key Lesson Learned

When multiple users in the same physical location experience connectivity issues, determine the scope of the outage first. Infrastructure components shared by all affected users should be investigated before troubleshooting individual workstations.
